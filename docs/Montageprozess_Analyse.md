# Montageprozess - IST-Analyse

**Erstellt:** 2025-12-11 | **Quelle:** Gespraech mit Andreas

---

## Prozess-Uebersicht

```
WARE KOMPLETT (aus Bestellprozess)
│
▼
1. PLANUNG
├─ Wer: Susann (hauptsaechlich), Rueckfragen an Enrico
├─ Team-Zuweisung: Nach Faehigkeiten der Monteure
├─ Tool: Outlook-Kalender (Ziel: W4A)
│
▼
2. VORBEREITUNG (Vorabend/Morgens)
├─ Susann: Unterlagen zusammenstellen
├─ Mario/Andreas/Jaroslaw: Material kommissionieren
├─ Problem: Oft falsche Dokumente!
│
▼
3. MORGEN-BRIEFING
├─ Wer: Jaroslaw + Andreas
├─ Dauer: 15-30 Min je nach Komplexitaet
│
▼
4. ANFAHRT + DURCHFUEHRUNG
├─ Arbeitszeit: 7-16 Uhr (aus Lager ab ~8 Uhr)
├─ Dauer: 1h bis mehrere Tage
├─ Kunde unterschreibt Montageauftrag
│
▼
5. RUECKMELDUNG
├─ Papier-Zettel (handschriftlich!)
├─ Arbeitszeit, Restarbeiten, Besonderheiten
│
▼
6. NACHBEREITUNG
├─ Ohne Restarbeiten → Rechnung erstellen
├─ Mit Restarbeiten → Kalender → neu verplanen
│
▼
7. RESTARBEIT (10-30% der Auftraege)
├─ Neuer Termin wenn Material/Voraussetzung da
└─ Teilmontagen moeglich
```

## Details pro Schritt

### 1. Teams

| Team | Personen | Fahrzeug | Bemerkung |
|------|----------|----------|-----------|
| Team 1 | Mariusz + Manfred | Auto 1 | Standard 2-Mann |
| Team 2 | Christian + Michael | Auto 2 | Standard 2-Mann |
| Team 3 | Stefan | Auto 3 | Meist allein, ausser bei schweren Teilen |

### 2. Planung

| Aspekt | IST-Zustand |
|--------|-------------|
| **Hauptverantwortlich** | Susann |
| **Rueckfragen** | An Enrico |
| **Entscheidungen** | Jaroslaw + Andreas |
| **Team-Zuweisung** | Susann, Enrico, Jaroslaw oder Andreas |
| **Kriterium** | Nach Faehigkeiten der Monteure |
| **Tool** | Outlook-Kalender |
| **Ziel** | W4A einbinden (Ansaetze fehlen noch) |

**Outlook-Kalender Struktur:**
| Bereich | Inhalt |
|---------|--------|
| **Oben** | Montage-Termine als farbige Kacheln (Kunde, Tel, Adresse) |
| **Kategorien unten** | Restarbeiten, Montage DKF, Mon: IT, Montage sonstiges, Montage HT, Servicearbeiten 1 Mann |
| **Rechte Spalte** | Bestellen, Noch in Klaerung, Abholung, Kunde meldet sich |
| **Info-Leiste** | Montagehilfe, Krankmeldungen |
| **Farbkodierung** | Verschiedene Farben je Montage-Typ |

**Erkenntnis:** Outlook wird als Planungs-Dashboard missbraucht - eigentlich zu komplex fuer einen Kalender!

### 3. Vorbereitung

**Unterlagen fuer Monteur:**
- Montageauftrag (2-seitig)
- Aufmassblaetter
- Lieferscheine (oft unnoetig)
- Bilder in gedruckter Form
- **Problem:** Susann stellt manuell zusammen, oft falsche/zu viele Dokumente

**Material-Vorbereitung:**
| Wer | Was |
|-----|-----|
| Mario (wenn im Lager) | Grosse Teile/Paletten vorholen |
| Andreas/Jaroslaw | Lagerware vorab kommissionieren |
| Morgens | Rest wird zusammengesucht |

### 4. Morgen-Briefing

| Aspekt | IST-Zustand |
|--------|-------------|
| **Wer leitet** | Jaroslaw + Andreas |
| **Frequenz** | Taeglich |
| **Dauer** | 15-30 Minuten je nach Komplexitaet |

### 5. Durchfuehrung

| Aspekt | IST-Zustand |
|--------|-------------|
| **Arbeitszeit** | 7-16 Uhr |
| **Start aus Lager** | Gegen 8 Uhr |
| **Dauer vor Ort** | Sehr unterschiedlich (1h bis mehrere Tage) |
| **Unterschrift** | Ja, Kunde unterschreibt auf Montageauftrag |
| **Werkzeug** | Gehoert dem Betrieb |

### 6. Krankmeldung

| Situation | Ablauf |
|-----------|--------|
| Standard | Termin faellt aus |
| Ausnahme | Jemand springt kurz fuer schwere Teile ein |
| Danach | Kann allein weiter montiert werden |

### 7. Rueckmeldung

| Aspekt | IST-Zustand |
|--------|-------------|
| **Medium** | Papier-Zettel (handschriftlich!) |
| **Inhalt** | Arbeitszeit, Restarbeiten, Besonderheiten |
| **Weg** | Wird spaeter ins Buero gebracht |
| **Problem-Prioritaet** | Doppelarbeit > Entzifferung > Verlust |

### 8. Zeiterfassung

| Art | Erfassung |
|-----|-----------|
| **Projekt-Zeit** | Auf Zettel, haendisch ausrechnen fuer Rechnung |
| **Arbeitszeit Lager** | Stempeluhr |
| **Arbeitszeit Buero** | W4A Zeiterfassung |

### 9. Rechnung

- Wird erstellt wenn Montageauftrag **ohne Restarbeiten** zurueckkommt

### 10. Restarbeiten (10-30% der Auftraege)

| Situation | Ablauf |
|-----------|--------|
| Nur Lagermaterial noetig | Auf Zettel notiert, schnell hinfahren wenn von aussen machbar |
| Sonst | Zettel ins Buero, im Kalender eintragen, wieder verplanen |
| Material muss bestellt werden | Kachel bleibt auf "Nicht geliefert" bis alles/wichtigstes da |
| Teilmontage | z.B. Fenster → warten auf Verputz → dann Raffstore messen/bestellen |

**Typische Gruende fuer Restarbeiten:**
- Unvorhergesehenes (Bauliches)
- Zeitproblem (nicht mehr geschafft)
- Material fehlt/falsch

## Prozess-Uebergaenge

### Bestellung → Montage

| Aspekt | IST-Zustand |
|--------|-------------|
| **Trigger** | Alle Lieferungen da ODER alle Teilprozess-Lieferungen bei abschnittsweiser Montage |
| **Entscheider (abschnittsweise)** | Immer Projektleiter (Jaroslaw/Andreas) |
| **Info-Kette** | Roland → Eingangslieferschein W4A → Info an Susann → Montage planbar |
| **Kacheln-Pflege** | Besteller pflegt, Susann schaut mit drueber |

### Montage → Bestellung (Rueckkopplung)

| Situation | Ablauf |
|-----------|--------|
| Kleinigkeiten | Zettel kommt rein |
| Gravierende Probleme | Anruf, Loesung vor Ort suchen |
| Teilmontage-Ausloeser | Kunde meldet Fortschritt (z.B. Verputz fertig) |
| Lange Spannen | Rueckfrage beim Kunden zum Fortschritt |
